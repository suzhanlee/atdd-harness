# Block 5: 최종 완성 + 실행

## EXPLAIN

### 지금까지 만든 것 정리

Block 0부터 여기까지, 단계별로 하나의 스킬을 완성해왔다:

```
Block 0: 도구 선택 + 스킬 골격 생성
  → 어떤 소스에서 정보를 모을지 정하고, 스킬 파일을 만들었다

Block 1: 프로젝트 구조 파악 + 스킬 맥락 반영
  → Explore 에이전트로 프로젝트를 살펴보고, 스킬에 맥락을 반영했다

Block 2: 도구 연결 (MCP/API)
  → 외부 서비스와 Claude를 연결하는 통로를 만들었다

Block 3: 수집 실행 & 검증
  → subagent로 여러 소스에서 동시에 정보를 수집하고 품질을 검증했다

Block 4: 출력 형식 설정
  → Markdown, Slack, Notion 중 출력 방식을 선택했다
```

### 이제 할 일

1. 완성된 스킬을 처음부터 끝까지 **실제로 실행**해서 결과를 확인한다
2. 스킬의 트리거를 점검한다 - 어떤 말을 하면 이 스킬이 작동하는지 확인
3. 원한다면 스킬 이름을 자신만의 이름으로 변경할 수 있다

### 트리거란?

스킬 파일 맨 위에 있는 `description`과 `triggers`가 트리거다:

```yaml
---
name: my-context-sync
description: 나의 컨텍스트 싱크. "싱크", "sync" 요청에 사용.
triggers:
  - "싱크"
  - "sync"
---
```

이 문구들 중 하나를 Claude에게 말하면, Claude가 알아서 이 스킬을 실행한다. 마치 "싱크해줘"라고 말하면 자동으로 수집이 시작되는 것이다.

## EXECUTE

3단계로 최종 완성과 실행을 진행한다:

**1단계: 최종 스킬 파일 확인**

사용자의 스킬 파일(`.claude/skills/my-context-sync/SKILL.md`)을 읽어서 전체 구성을 한 번 더 확인한다. 주요 점검 항목:

- 소스 정의가 올바른지 (연결한 도구와 일치하는지)
- 실행 흐름이 맞는지 (병렬 수집 → 통합 → 저장 순서)
- 출력 포맷이 Block 4에서 선택한 대로 설정되어 있는지
- 트리거 문구가 적절한지

문제가 있으면 함께 수정한다.

**2단계: 스킬 실행**

실제로 스킬을 실행한다. 사용자에게 아래와 같이 안내한다:

```
이제 완성된 스킬을 실행해봅시다!
새 대화를 열고 아래처럼 입력해보세요:

  싱크해줘

또는 슬래시 명령어로:

  /my-context-sync
```

> Claude가 직접 실행하여 결과를 보여줄 수도 있다. 사용자가 원하면 Claude가 바로 실행한다.

**3단계: 결과 확인 + 마무리**

실행 결과를 함께 확인한다:
- Markdown 파일이 정상적으로 생성되었는지
- Slack 전송을 선택했다면 메시지가 도착했는지
- Notion을 선택했다면 페이지가 생성되었는지
- 수집된 내용이 기대한 대로인지

문제가 있으면 함께 수정하고 다시 실행한다.

**성공했다면 축하 메시지와 활용 팁을 전달한다:**

```
축하합니다! 나만의 Context Sync 스킬이 완성되었습니다!

이제 "싱크해줘" 한마디로 흩어진 정보가 자동으로 모입니다.
Day 1에서 Claude Code를 처음 만졌는데, 벌써 나만의 자동화 스킬을 만들었습니다.

활용 팁:
  1. 매일 아침 실행하면 "모닝 브리핑"처럼 사용할 수 있습니다
  2. CLAUDE.md에 스케줄을 추가하면 매일 자동 실행도 가능합니다:
     ## 스케줄
     - context-sync: 매일 09:00 실행
  3. 소스를 추가하고 싶으면 스킬 파일에 같은 형식으로 넣으면 됩니다
  4. Day 3 이후에 이 스킬을 더 발전시킬 수 있습니다
```
