# ADR Self-Critique Guide

## 개요

Self-Critique는 작성한 ADR을 스스로 비평하여 품질을 높이는 과정이다.
자기 성찰을 통해 문서의 약점을 스스로 발견하고 보완한다.

---

## Self-Critique 체크리스트 (Phase D)

각 항목을 1~5점으로 평가한다.

### 1. Context 충분성

**질문**: 6개월 후 다른 사람이 이 ADR을 읽어도 결정 배경을 완전히 이해할 수 있는가?

| 점수 | 기준 |
|------|------|
| 1점 | 배경 설명이 거의 없음 |
| 2점 | 일부 배경만 설명됨 |
| 3점 | 기본적인 배경은 있으나 맥락 부족 |
| 4점 | 대부분의 배경이 설명됨 |
| 5점 | 누가 읽어도 완전히 이해 가능 |

**평가 포인트**:
- [ ] 비즈니스 요구사항이 명시되어 있는가?
- [ ] 기술적 제약사항이 구체적으로 기술되어 있는가?
- [ ] 당시 팀/조직 상황이 설명되어 있는가?
- [ ] 왜 지금 결정해야 하는지 이유가 있는가?

---

### 2. 대안 분석 깊이

**질문**: 정말 최선의 선택인가? 다른 대안은 충분히 고려했는가?

| 점수 | 기준 |
|------|------|
| 1점 | 대안 없이 결정만 기술 |
| 2점 | 1개 대안만 형식적으로 언급 |
| 3점 | 2개 대안이 있으나 깊이 분석 안 됨 |
| 4점 | 3개 이상 대안 분석, 일부 깊이 부족 |
| 5점 | 3개 이상 대안을 깊이 있게 분석, Trade-off Matrix 완성 |

**평가 포인트**:
- [ ] 최소 3개 이상의 대안이 고려되었는가?
- [ ] 각 대안의 장단점이 구체적으로 분석되었는가?
- [ ] "선택하지 않은 이유"가 논리적인가?
- [ ] Trade-off Matrix가 작성되었는가?

---

### 3. Consequences 솔직성

**질문**: 부정적 결과와 위험까지 솔직하게 포함했는가?

| 점수 | 기준 |
|------|------|
| 1점 | 긍정적 결과만 기술 |
| 2점 | 부정적 결과가 있으나 모호함 |
| 3점 | 일부 부정적 결과 포함, 위험은 누락 |
| 4점 | 부정적 결과와 위험 대부분 포함 |
| 5점 | 긍정/부정/위험이 균형 있게 기술됨 |

**평가 포인트**:
- [ ] 부정적 결과가 구체적으로 기술되어 있는가?
- [ ] 잠재적 위험이 식별되어 있는가?
- [ ] 위험 완화 전략이 포함되어 있는가?
- [ ] "언제 이 결정이 틀릴 수 있는가"가 설명되어 있는가?

---

### 4. Reconsideration 구체성

**질문**: 재검토 조건이 명확하고 측정 가능한가?

| 점수 | 기준 |
|------|------|
| 1점 | 재검토 조건 없음 |
| 2점 | "필요 시 재검토" 등 모호한 조건 |
| 3점 | 일부 조건이 있으나 측정 불가 |
| 4점 | 구체적 조건이 있으나 정량적이지 않음 |
| 5점 | 정량적 임계값으로 재검토 조건 명시 |

**평가 포인트**:
- [ ] 재검토 트리거가 구체적인가?
- [ ] 정량적 임계값이 있는가? (예: "DAU 10만 돌파 시")
- [ ] 재검토 시점이 정의되어 있는가?
- [ ] 이 결정을 대체할 후속 ADR 계획이 있는가?

---

### 5. 설득력

**질문**: 반대 의견에 논리적으로 대응할 수 있는가?

| 점수 | 기준 |
|------|------|
| 1점 | 결정에 대한 근거 없음 |
| 2점 | 근거가 있으나 약함 |
| 3점 | 기본적 근거는 있으나 반론에 취약 |
| 4점 | 타당한 근거, 일부 반론 대응 가능 |
| 5점 | 강력한 근거, 예상 반론까지 선제 대응 |

**평가 포인트**:
- [ ] 결정의 논리적 근거가 명확한가?
- [ ] 예상되는 반대 의견이 무엇인가?
- [ ] 반대 의견에 대한 대응 논리가 있는가?
- [ ] 이해관계자의 우려사항이 다뤄졌는가?

## Self-Critique 점수표 Template

```markdown
## Self-Critique Score

| # | 항목 | 점수(1~5) | 개선 필요 사항 |
|---|------|-----------|----------------|
| 1 | Context 충분성 | | |
| 2 | 대안 분석 깊이 | | |
| 3 | Consequences 솔직성 | | |
| 4 | Reconsideration 구체성 | | |
| 5 | 설득력 | | |
| **총점** | | **/25** | |

### 개선 계획
- [평균 4점 미만 항목 수정 필요]
```

## 평가 기준

### 총점 해석

| 총점 | 등급 | 설명 |
|------|------|------|
| 22-25 | A | 우수, 바로 Accept 가능 |
| 18-21 | B | 양호, 소수 항목 보완 권장 |
| 14-17 | C | 보통, 여러 항목 보완 필요 |
| 10-13 | D | 미흡, 전면 수정 권장 |
| 5-9 | F | 부족, 처음부터 다시 작성 |

### 평균 기준

**평균 4점 미만 항목은 반드시 수정이 필요하다.**

```
평균 = 항목 점수 합계 / 5
예: (5 + 3 + 4 + 2 + 4) / 5 = 3.6 → 수정 필요
```

## Self-Critique 예시

### 예시: ADR-001 MySQL 선택

```markdown
## Self-Critique Score

| # | 항목 | 점수 | 비고 |
|---|------|------|------|
| 1 | Context 충분성 | 4 | 비즈니스 요구사항 구체적, 팀 상황 설명됨 |
| 2 | 대안 분석 깊이 | 3 | PostgreSQL, MongoDB만 고려, 다른 RDBMS 누락 |
| 3 | Consequences 속직성 | 5 | 긍정/부정/위험 모두 상세히 기술 |
| 4 | Reconsideration 구체성 | 2 | "필요 시 재검토"만 있어 측정 불가 |
| 5 | 설득력 | 4 | 근거 타당, 예상 반론 일부 대응 |
| **총점** | | **18/25** | **평균 3.6** |

### 개선 계획
- [ ] 항목 2: MariaDB, CockroachDB 대안 추가 분석
- [ ] 항목 4: "DAU 50만 돌파 시" 또는 "단일 테이블 1억 건 초과 시" 재검토 조건 명시
```

## Self-Critique 수행 팁

### Before Self-Critique
1. ADR 작성 직후 바로 수행하지 말고, 최소 30분 후 수행
2. "비평가" 관점에서 읽는다고 가정
3. "내가 이 결정에 반대한다면?"을 생각하며 읽기

### During Self-Critique
1. 각 질문에 솔직하게 답변
2. 낮은 점수 항목은 구체적 개선 방안 제시
3. "충분하다"고 스스로 판단하지 말고 객관적 기준 적용

### After Self-Critique
1. 평균 4점 미만 항목은 반드시 수정
2. 수정 후 Self-Critique 재수행
3. `/redteam` 실행 전 Self-Critique 완료 필수
